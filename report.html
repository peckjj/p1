RYSA
Rockville Youth Soccer Association

Project 1 report:

Itemized list:

  1. The client is a local recreational sports league with need for a main website for league and registration information.
  2. All images are used in accordance with copyright law. The RYSA logo was carefully and meticulously crafted in an online wordart generator.
  3. The links to all five pages are found in the navbar above, the Home link navigates to index.html.
  4. The RYSA header is found above the main content of each page. It contains the RYSA logo.
  5. Navigation links are located at the top of each page.
  6. Media queries located in ./css/style.css control layout for small screens. They are also present in other css files for specific modifications.
  7. Layout across the site does not change, except in teams.html. This page intentionally uses a slightly unique format. The color scheme and aesthetic styles are uniform across the site.
  8. All images have a corresponding 'alt' attribute.
  9. All pages declare and use the same favicon.
  10. Directory organization meets requirements. All non-HTML files are in appropriate subfolders.
  11. Only HTML and CSS has been used to create this website.
  12. Each page passes the Nu Validator.
  13. All text content aligns to a strong left axis, except in the case of teams.html, where center alignment was explicitly chosen.
  14. Various fonts are used across the site. The recommendation for headers and body content were followed, with some unique fonts used in teams.html.
  15. Two forms are found in signup.html. They are intended to send registration information to a server.
  16. Links to external sites are found on the Home page, or index.html, where users can find additional information relating to a particular news post.
  17. Common naming and formatting conventions were followed
  18. Upon loading, the main content of each page performs a brief expanding animation. Content in teams.html also interactively illuminates.
  19. The Git repo can be located here.
  20. All file names are alphanumeric and lowercase.
  21. The site's domain is "project1.peck-projects.me".
  22. Secure HTTP has been implemented.
  23. Port 80 redirects to port 443.
  24. Unfortunately, Git commits were not steady. Much work was done in a local environment before a majority of the content was uploaded.

Client Critique

A discussion via Zoom was held with the client. During the conversation, the client stated various pros and cons:

pros:

  • The muted color scheme and simple layout provides a user-friendly experience.
  • The layout of the teams page is vibrant, and let's users get an impression of each team.

cons:

  • The teams page should include another layer of organization to divide teams into age groups.
  • The forms contain all necessary information, but could include another field for any additional information
